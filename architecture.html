Clipscript · How it works

Every word, lifted from the video.

Paste a link from YouTube, TikTok, or Instagram. Clipscript pulls the full transcript — captions when the video already has them, a clean audio transcription when it doesn’t — and keeps every one you’ve made.

Operational NestJS Next.js PostgreSQL yt-dlp ffmpeg OpenAI

The pipeline

The journey of a link

One paste kicks off a chain that runs on its own. The link is checked, the video is reached, and Clipscript takes the fastest route to the words — rejoining at the same place no matter which path it took.

The fork

Two ways to a transcript

Captions are free and fast, so Clipscript always tries them first. Only when a video has none does it do the heavier work of listening to the audio.

Path one · captions

Straight from the source

Almost every YouTube video

  1. Pull the caption track with the video’s details
  2. Parse it into clean, readable text
  3. Done — no download, no transcription
≈ 40 seconds · no cost

Path two · audio

Listen, then write it down

TikTok, Instagram & caption-less video

  1. Download the audio and convert it to mp3
  2. Send it to OpenAI for transcription
  3. Store the finished text
a few minutes · capped at 30 min

The system

What talks to what

A Next.js page in the browser, a NestJS backend that runs the pipeline, one Postgres table, and the local tools it drives to reach each platform.

Live today

What’s working

Every capability below is running and has been used against real videos — the three platforms transcribe, and the pipeline recovers from its own edges.

The record

One row per transcript

There's no login, so the server keeps only public content: this one table is a cache of transcripts keyed by the normalized link, with no notion of who made them. Your own history lives in your browser, not here. The normalized link stops the same public video being processed twice.
